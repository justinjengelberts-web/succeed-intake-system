Architecture Overview

~1 minute
Show: index.ts in editor

"The system is a single Supabase Edge Function running a 5-step pipeline:"

Walk through each step

  • Extract — Claude parses the raw message into structured data
  • Score — deterministic TypeScript engine, no AI, scores 0 to 100
  • Route — maps score to action: schedule call, request info, manual review, or archive
  • Follow-up — Claude drafts a response (only if score >= 20, spam gets nothing)
  • Output — writes everything to Google Sheets, every step logs to Supabase

Slide 8 of 9

Design Trade-Off: Deterministic Scoring

~1 minute
Show: score.ts in editor

"The key trade-off: I deliberately kept scoring out of the LLM. It would have been faster to ask Claude to score everything in one pass, but then you can't explain why something scored 87."

Point out in the code

  • 4 weighted dimensions: completeness, value signals, platform fit, intent clarity
  • Every point maps to a specific factor: +10 high fee, +12 age range fits, +6 institutional email
  • That's auditable and tunable. If Succeed's priorities change, you adjust a rule, not a prompt
  • LLM is used where it adds value: understanding messy language (extraction) and writing natural responses (follow-ups)
